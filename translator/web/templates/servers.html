How to start a server

Run on the machine with the GPU (Mac or Windows):

Recommended Reverse registration + pull mode — only outbound connections from the Mac to this host. Works even if Mac is on a different subnet (WiFi vs wired), no port forwarding needed on the Mac. 

# On Mac/remote server — replace HOST_IP with this Windows machine's IP
python server.py --host 0.0.0.0 --port 8765 --host-url http://HOST_IP:5000

# The remote will:
# 1. Register itself with the host (POST /api/workers/register)
# 2. Send heartbeats every 15s (POST /api/workers/heartbeat)
# 3. Poll the host for translation chunks (GET /api/workers/{label}/chunk)
# 4. Post results back (POST /api/workers/{label}/result)
# → Mac only makes outbound connections to HOST_IP — no port forwarding on Mac needed

# Other options:
python server.py --model-path /path/to/model.gguf  # override model path
python server.py --gpu-layers 0                     # CPU only
python server.py --no-mdns                          # disable mDNS announcement

For legacy direct connection (same subnet only):

remote:
  mode: remote
  server_url: "http://192.168.1.x:8765"  # host connects to remote directly
